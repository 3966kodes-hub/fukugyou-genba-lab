「SEO で上位表示する記事を書く」のは、勘や努力ではなく **「再現可能なプロセス」**で達成できます。本記事で 7 ステップの標準フローを解説。

## 🎯 SEO 記事執筆の 7 ステップ全体像

| ステップ | 所要時間 | 目的 |
|---|---|---|
| 1. KW 選定 | 30 分 | 狙う検索キーワードを決める |
| 2. 競合分析 | 30 分 | 上位記事を分析して差別化点を見つける |
| 3. 構成作成 | 30 分 | H2/H3 を設計 |
| 4. 本文執筆 | 2-3 時間 | 5,000-8,000 字 |
| 5. 校正 | 30 分 | 誤字・重複・読みやすさチェック |
| 6. 公開 | 30 分 | アイキャッチ・メタ情報・内部リンク |
| 7. リライト | 月次 | データに基づく改善 |

合計 **約 5 時間 / 記事**（慣れれば 3 時間）。

## ステップ 1：KW 選定

### KW 選定の 3 軸

| 軸 | 確認方法 |
|---|---|
| **検索ボリューム** | ラッコキーワード / Ubersuggest |
| **競合の強さ** | 検索結果 1 ページ目を見る |
| **収益性** | ASP に該当案件あるか |

### KW タイプ別の戦略

| KW タイプ | 例 | 狙い |
|---|---|---|
| ビッグ KW | 「副業」 | ❌ 個人では無理 |
| ミドル KW | 「副業 おすすめ」 | 🟡 競合厳しい |
| ロングテール KW | 「副業 おすすめ 在宅 月5万」 | 🟢 個人で勝てる |

→ **個人ブログはロングテール一択**。3〜4 語の組み合わせ KW を狙う。

## ステップ 2：競合分析

検索結果 1 ページ目（10 記事）を全部開く：

| チェック項目 | 内容 |
|---|---|
| タイトル傾向 | 数字 / 感情ワード使われている？ |
| 文字数 | 平均何字？ |
| 構成（H2 数）| 何セクションで構成されている？ |
| 含まれる情報 | 共通項 + 抜けている観点を発見 |
| 上位記事の弱み | 古い情報 / 浅い分析 / 体験談なし 等 |

→ 「上位記事の弱みを埋める + 独自視点を追加」が SEO の本質。

## ステップ 3：構成作成

H2/H3 を設計：

```
H1: タイトル（KW 含む）
　H2: 概要セクション
　H2: 詳細セクション 1
　　H3: 個別ポイント A
　　H3: 個別ポイント B
　H2: 詳細セクション 2
　H2: 比較・FAQ
　H2: まとめ
```

→ 上位記事が 5 セクションなら、**自分は 7 セクション**で網羅性アピール。

## ステップ 4：本文執筆

### 文字数の目安

| ジャンル | 目安 |
|---|---|
| ビギナー向け解説 | 5,000-8,000 字 |
| 比較記事 | 6,000-10,000 字 |
| 体験記 | 3,000-5,000 字 |
| ニュース・速報 | 1,500-3,000 字 |

### 読みやすさのコツ

- 1 段落 = 3〜5 行
- 結論ファースト
- 表 / 箇条書き多用
- 専門用語はかみ砕いて説明
- 画像 / 図解を 1 セクションに 1 個

## ステップ 5：校正

| チェック項目 | ツール |
|---|---|
| 誤字脱字 | Word 校正機能 / Grammarly |
| 表現重複 | 文賢 / 自分で読み直し |
| 読みやすさ | 音読 / Hemingway Editor |
| AI 校正 | ChatGPT / Claude で品質チェック |

→ **公開前に必ず音読**。違和感のある文章は読者にも違和感を与える。

## ステップ 6：公開

### 公開前チェックリスト

- [ ] タイトルに KW 含む
- [ ] メタディスクリプション 100-120 字
- [ ] アイキャッチ画像（1200×630）
- [ ] H2/H3 構造正しい
- [ ] 内部リンク 3 件以上
- [ ] 外部リンク（出典）
- [ ] アフィリエイトリンクに PR 表記
- [ ] パーマリンク（URL）が KW 含む
- [ ] カテゴリ + タグ設定

## ステップ 7：リライト

公開 1〜3 ヶ月後、Google Search Console で：

| データ | アクション |
|---|---|
| 表示回数多い + クリック率低い | タイトル / メタ改善 |
| クリック率高い + 順位 11-30 位 | 内容深堀り → 上位狙い |
| アクセス減少傾向 | 古い情報更新 + 追記 |
| 全くアクセスない | 別 KW で書き直し |

→ **書きっぱなしは NG**。月次でリライト PDCA を回す。

## 🤖 AI 活用フロー

| ステップ | AI 活用 |
|---|---|
| KW 選定 | ChatGPT で関連 KW 一括出し |
| 競合分析 | Perplexity / NotebookLM で要約 |
| 構成 | Claude で H2/H3 案出し |
| 本文 | 自分で執筆 + AI 校正 |
| 校正 | Claude / ChatGPT 双方でチェック |

<Callout type="warning" title="AI 100% 生成は危険">
Google は **「AI 量産記事のペナルティ」**を強化中。
**「AI で構成支援 + 自分で執筆」**が安全で長期収益化に強い。
</Callout>

## まとめ：再現可能なプロセスで書く

SEO 記事執筆は才能ではなく **「再現可能なプロセス」**。7 ステップを徹底すれば、3 ヶ月後には誰でも一定品質の記事が量産できます。
